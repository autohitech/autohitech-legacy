# 데이터베이스 정보: autotech

이 문서는 Autohitech 데이터베이스의 테이블들이 어떤 기능을 수행하는지, 그누보드 4 표준 모듈과 커스텀 비즈니스 모듈로 분류하여 설명합니다.

## 🏢 비즈니스 및 CRM 모듈 (커스텀)

Autohitech 시스템 고유의 테이블들로, 내부 비즈니스 프로세스를 관리합니다.

| 테이블 | 목적 | 설명 |
| :--- | :--- | :--- |
| **`diary`** | **업무 일지 (Daily Report)** | 가장 많은 데이터(22,000건 이상)가 포함된 핵심 테이블입니다. 일일 현장 활동, 사이트 방문, 기술 지원 로그(예: PLC 점검, 사이트 조사)를 기록하는 데 사용됩니다. |
| **`customers`** | **고객/거래처 관리** | 업체명, 담당자명, 전화번호, 주소, 이메일 등 비즈니스 파트너와 고객 정보를 관리하는 CRM 성격의 테이블입니다. |
| **`adm_user`** | **관리자 사용자 관리** | 보조 관리자 인증 테이블 또는 커스텀 모듈 전용 권한 관리용으로 보입니다. |
| **`bbs`** | **레거시 게시판** | 오래된 또는 간소화된 게시판 구조로, 내부 메모나 과거 데이터 보관용으로 보입니다. |
| **`Sheet1`** | **임시 데이터 임포트** | 일회성 데이터 마이그레이션 또는 참조를 위해 엑셀 등에서 가져온 시트입니다. |

## 🛠️ 그누보드 4 핵심 테이블 (표준)

웹사이트의 커뮤니티 기능과 기본 설정을 담당하는 표준 테이블들입니다.

### 1. 설정 및 관리
- **`g4_config`**: 사이트 제목, 테마, 관리자 이메일 등 전역 설정을 저장합니다.
- **`g4_group`**: 여러 게시판을 논리적 그룹으로 분류합니다.
- **`g4_auth`**: 특정 모듈에 대한 세부적인 관리자 권한을 관리합니다.
- **`g4_banner`**: 롤링 배너나 광고 이미지를 관리합니다.

### 2. 회원 및 계정
- **`g4_member`**: 핵심 회원 정보(ID, 비밀번호, 닉네임, 레벨, 포인트 등)를 저장합니다.
- **`g4_login`**: 사용자의 로그인 기록과 IP 주소를 추적합니다.
- **`g4_memo`**: 회원 간에 주고받은 내부 쪽지 정보를 저장합니다.
- **`g4_point`**: 포인트 획득/사용 내역을 기록하는 장부입니다.
- **`g4_scrap`**: 사용자가 프로필에 스크랩한 게시물 정보를 저장합니다.

### 3. 게시판 및 컨텐츠
- **`g4_board`**: 개별 게시판의 설정(권한, 스킨 등)을 저장합니다.
- **`g4_board_file`**: 게시물에 업로드된 모든 파일의 메타데이터를 저장합니다.
- **`g4_board_new`**: "최근 게시물" 위젯을 위해 모든 게시판의 최신글 인덱스를 저장합니다.
- **`g4_write_[bo_table]`**: 각 게시판의 실제 게시물 내용이 저장되는 테이블입니다 (예: `g4_write_notice`).

### 4. 통계 및 추적
- **`g4_visit`**: 실시간 방문자 수와 통계(OS, 브라우저, 유입 경로)를 저장합니다.
- **`g4_popular`**: 자주 사용되는 검색어와 검색 횟수를 저장합니다.
- **`g4_homeconfig`**: 홈페이지 레이아웃이나 디자인 요소에 대한 커스텀 설정을 저장합니다.

## 📝 참고 사항
- **G4 접두사**: 모든 표준 그누보드 4 테이블은 `g4_`로 시작합니다.
- **백업 테이블**: `customers22`, `customers-0305`, `customers_old` 등은 메인 `customers` 테이블의 백업 또는 과거 스냅샷 버전입니다.
- **인코딩**: `diary`와 `customers`의 일부 데이터는 원본 SQL에서 인코딩 이슈가 보일 수 있으나, 웹 인터페이스에서는 한국어(EUC-KR/UTF-8)로 정상 출력되도록 설계되었습니다.
